# Aura Shell — Phase 0: Промпты для Cursor AI

## Обзор

Phase 0 разбита на **8 последовательных промптов**. Каждый промпт:
- Выполняется в **одном контексте** Cursor (один чат/сессия).
- Создаёт конкретные `.asm` файлы и тесты.
- Обновляет `TODO.md` (отмечает выполненное).
- Дополняет `REPORT.md` (что сделано, проблемы, решения).

## Порядок выполнения

```
STEP_01 → STEP_02 → STEP_03 → STEP_04 → STEP_05 → STEP_06 → STEP_07 → STEP_08
  HAL      Memory    Threads    Event     Canvas     Wayland    Input      REPL
```

Каждый шаг зависит от предыдущих. Не перескакивайте.

## Как использовать

1. Создайте репозиторий `aura-shell/` со структурой из ТЗ.
2. Поместите файлы `TODO.md`, `REPORT.md` и все `STEP_XX.md` в корень репозитория.
3. Откройте Cursor, загрузите `STEP_01.md` в контекст.
4. Скопируйте содержимое промпта в Cursor AI.
5. После выполнения — проверьте `REPORT.md`, отправьте его на ревью.
6. После ревью — переходите к `STEP_02.md`.

## Файлы

| Файл | Назначение |
|---|---|
| `TODO.md` | Общий чеклист Phase 0, обновляется после каждого шага |
| `REPORT.md` | Накопительный отчёт, дополняется после каждого шага |
| `STEP_01.md` — `STEP_08.md` | Промпты для Cursor AI |

## Критерий завершения Phase 0

Phase 0 считается завершённой когда:
- Все пункты в `TODO.md` отмечены `[x]`.
- `REPORT.md` содержит записи по всем 8 шагам.
- Бинарник `aura-shell` запускается на Linux x86_64.
- Открывается окно (Wayland client).
- Курсор мигает, текст вводится с клавиатуры и отображается.
- Touch-события и mouse-события обрабатываются единообразно.
